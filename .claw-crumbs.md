- **Prod-login HARD gotcha (2026-06-08):** the two `NEXT_PUBLIC_*` Supabase vars are
  **inlined into the browser bundle at BUILD time** ‚Äî they must exist in Vercel **Production**
  scope and any change needs a **fresh redeploy** (`--force` / uncheck build cache) to take.

  "Something went wrong". Symptom-2: `signInWithPassword` ‚Üí `TypeError: Failed to execute
  'fetch': Invalid value` means a **malformed header value** ‚Äî caused here by pasting the
  anon key from the Supabase dashboard **with the trailing newline + "This key is safe‚Ä¶"
  caption text** appended; the `Authorization: Bearer <key>\nThis key‚Ä¶` header is illegal.
  Fix: env value must be the bare JWT only. Verify a deploy by curling the live bundle and
  grepping for `jpzkscgwupdkbhlauqmf.supabase.co` + a clean anon JWT (no trailing chars).
- **Auth:** invite-only Supabase Auth. `role` + `applicant_id` in `app_metadata`
  (service-role only). Admin invites / password-reset / delete from `/admin`.
- **Email:** Supabase custom SMTP via **Resend** (configured in the Supabase **dashboard**,

## Standing rules (cause real bugs if forgotten)
- **Real PII discipline:** `CMOP_Process.xlsx` = 44 real people. Analysis only ‚Äî never parse
  names/emails/phones into git, a committed seed, or `.data/`. Real ID PDFs/scans contain a
  real SSN ‚Äî gitignored; never sync/commit.
- Fill the **original VA AcroForm PDFs** via `pdf-lib` (lowest rejection risk); keep
  prefilled contract fields; leave signature/date blank for wet-signing.
- **Packet download gating:** `packetDownloadable` requires every form uploaded **+ stored**
  (real bytes on disk) **+ 100% + zero issues**, AND status REVIEWED. The displayed `%`
  (`packetCompleteness`) ignores `stored`/issues ‚Äî so a packet can read **100% yet not be
  downloadable**. Seed candidates are 100% but never `stored` ‚Üí no Download button until a
  real upload. Test helper: `node .qa-build/make-downloadable.mjs <applicantId>`.
